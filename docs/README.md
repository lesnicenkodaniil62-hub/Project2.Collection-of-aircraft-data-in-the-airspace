# **Сбор данных о воздушных судах в воздушном пространстве**

## Описание
Данный проект представляет собой модульное Python-приложение, разработанное с соблюдением принципов SOLID (в частности, SRP) и Defensive Programming. Архитектура разделена на чёткие слои: API-клиенты, строго типизированные модели данных с валидацией, абстрактные хранилища и утилиты обработки.
Ключевой особенностью является двухуровневая система хранения: рабочий файл (data/aeroplanes.json) и система снапшотов (data/aeroplanes_*.json), что позволяет безопасно экспериментировать с данными. Весь код строго типизирован и проходит проверку mypy --strict, а стиль соответствует PEP 8.

## Нововведение и функции:
###  Нововведение для пользователей1. Была добавлена функция 
1. Была введена функция система снапшотов: Каждое сохранение создаёт отдельный файл с временной меткой, предотвращая случайную перезапись важных данных.
2. Была введена функция интерактивный статус-бар: После каждого действия в консоли отображается актуальное состояние: Статус: В памяти: X | В файле: Y.
3. Была введена функция умный диалог слияния данных: При повторном запросе данных система спрашивает, заменить текущий набор или дополнить его.
4. Была введена функция защита от потери данных: При попытке выйти из программы или загрузить новый снапшот при наличии несохранённых изменений появляется предупреждение с запросом подтверждения.
5. Была введена функция улучшенный табличный вывод: Данные выводятся в виде выровненной таблицы с заголовками, единицами измерения и понятным статусом.
###  Нововведение технические не предназначенные для глаз пользователя
1. Была введена функция функциональное тестирование: Полный отказ от классов в тестах в пользу чистых функций с активным использованием @pytest.fixture и @pytest.mark.parametrize.
2. Была введена функция глубокая изоляция зависимостей: Повсеместное использование unittest.mock.patch и MagicMock для тестирования API и консольного ввода без реальных сетевых запросов.
3. Была введена функция строгая типизация: Внедрение mypy --strict на уровне конфигурации проекта.
4. Была введена функция устранение E731 (flake8): Замена анонимных lambda функций на именованные внутренние функции для улучшения читаемости.
5. Была введена функция устойчивость к повреждённым данным: Методы загрузки JSON перехватывают ошибки и автоматически инициализируют чистое хранилище.
 
Дальнейшие функции будут описаны в последующих обновлениях.

## **Примечание к функциям**
### Подробное описание всех функций проекта
#### Модуль src/api/base.py
* __init__: Инициализирует базовый URL, таймаут, User-Agent и сессию requests.
* _make_request: Выполняет HTTP-запрос с обработкой исключений (Timeout, HTTPError, RequestException, ValueError) и возвратом JSON или None.
* get_coordinates: Абстрактный метод для получения координат страны.
* get_aeroplanes: Абстрактный метод для получения данных о самолётах.
* get_all_states: Абстрактный метод для получения всех состояний ВС.
* __repr__: Возвращает строковое представление объекта для отладки.
#### Модуль src/api/nominatim.py
* __init__: Инициализирует клиент с URL Nominatim и строгим User-Agent.
* get_coordinates: Запрашивает координаты по названию страны, парсит ответ и возвращает кортеж (latitude, longitude) или None.
* get_bounding_box: Запрашивает и возвращает ограничивающий прямоугольник страны (west, east, south, north).
* get_aeroplanes: Заглушка, возвращает пустой список и логирует предупреждение.
* get_all_states: Заглушка, возвращает пустой список и логирует предупреждение.
#### Модуль src/api/opensky.py
* __init__: Инициализирует клиент, опционально настраивая базовую аутентификацию (username/password).
* get_coordinates: Заглушка, возвращает None и логирует предупреждение.
* get_all_states: Запрашивает все состояния ВС и парсит их через _parse_states.
* get_aeroplanes: Запрашивает состояния ВС с фильтром по стране регистрации.
* _parse_states: Преобразует "сырой" ответ OpenSky (список списков) в список словарей с именованными ключами, игнорируя некорректные записи.
* __repr__: Возвращает строковое представление, указывающее на наличие аутентификации.
#### Модуль src/models/aeroplane.py
* __post_init__: Автоматически вызывает метод validate после создания объекта.
* validate: Проверяет типы и диапазоны всех атрибутов (callsign, country, velocity, altitude, on_ground, icao24, latitude, longitude), вызывая TypeError или ValueError при ошибках, и логирует предупреждения при экстремальных значениях.
* compare_by_speed: Сравнивает текущий самолёт с другим по скорости, возвращая -1, 0 или 1.
* compare_by_altitude: Сравнивает текущий самолёт с другим по высоте, возвращая -1, 0 или 1.
* to_dict: Преобразует объект в словарь для сериализации в JSON.
* from_dict: Классовый метод для создания объекта Aeroplane из словаря с проверкой обязательных полей.
* cast_to_object_list: Классовый метод для массового преобразования списка словарей от API в список объектов Aeroplane, игнорируя записи с пустым callsign и обрабатывая None в числовых полях.
* __str__: Возвращает читаемое строковое представление для вывода в консоль.
* __repr__: Возвращает техническое строковое представление для отладки.
#### Модуль src/storage/json_saver.py
* __init__: Инициализирует путь к файлу и загружает существующие данные или создаёт новый файл.
* _load_from_file: Загружает данные из JSON, обрабатывая ошибки декодирования и создавая пустую структуру при сбое.
* _save_to_file: Сохраняет текущее состояние _data в рабочий JSON-файл.
* save_snapshot: Сохраняет переданный список самолётов в новый файл с уникальным именем, содержащим timestamp.
* list_snapshots: Возвращает отсортированный список путей ко всем файлам-снапшотам в директории.
* load_snapshot: Загружает данные из указанного файла-снапшота в рабочее хранилище.
* add_aeroplane: Добавляет один самолёт, обновляя запись, если callsign уже существует.
* add_aeroplanes: Добавляет список самолётов, пропуская дубликаты.
* get_aeroplanes: Возвращает список самолётов, отфильтрованных по переданным критериям (**criteria).
* _matches_criteria: Внутренний метод для проверки соответствия самолёта заданным критериям фильтрации.
* get_all_aeroplanes: Возвращает список всех самолётов из хранилища в виде объектов.
* delete_aeroplane: Удаляет самолёт по callsign, возвращая True при успехе.
* delete_aeroplanes: Удаляет несколько самолётов по критериям, возвращая количество удалённых записей.
* clear: Полностью очищает хранилище и перезаписывает файл пустым массивом.
* count: Возвращает текущее количество самолётов в хранилище.
* exists: Проверяет наличие самолёта с указанным callsign.
* __repr__: Возвращает строковое представление хранилища с путём и количеством записей.
#### Модуль src/utils/data_processing.py
* filter_aeroplanes: Фильтрует список самолётов по заданным критериям (страна, высота, скорость, статус).
* _get_sort_key: Внутренняя именованная функция для безопасного получения значения атрибута при сортировке (замена lambda).
* sort_aeroplanes: Сортирует список самолётов по указанному атрибуту в прямом или обратном порядке.
* get_top_aeroplanes: Возвращает топ-N самолётов, отсортированных по убыванию заданного параметра.
* get_aeroplanes_by_altitude_range: Обёртка над filter_aeroplanes для фильтрации строго по диапазону высот.
* print_aeroplanes: Выводит список самолётов в консоль в виде красивой таблицы с заголовками и разделителями.
* show_status_bar: Выводит в консоль текущее количество самолётов в памяти и в файле.
#### Модуль src/utils/user_interaction.py
* user_interaction: Главная функция, реализующая бесконечный цикл while True с обработкой пользовательского ввода, вызовом API, сохранением/загрузкой снапшотов и выводом статус-бара.
#### Модуль main.py
* setup_logging: Настраивает двойное логирование: в консоль (уровень INFO и выше) и в файл logs/application.log (уровень DEBUG и выше).
* print_banner: Выводит ASCII-арт приветствия в консоль.
* main: Точка входа в приложение. Инициализирует логи, выводит баннер, запускает user_interaction и обрабатывает KeyboardInterrupt и общие исключения.

## **Тестирование кодов**

### conftest.py (Глобальные фикстуры)

1. valid_aeroplane_data: Предоставляет эталонный словарь с валидными данными для создания самолёта в тестах. 
2. aeroplane: Создаёт и возвращает готовый валидный объект Aeroplane на основе фикстуры valid_aeroplane_data. 
3. aeroplane_list: Генерирует список из 4 разных самолётов (с разными странами, высотами, скоростями и статусом on_ground) для тестов фильтрации и сортировки. 
4. mock_raw_opensky_state: Предоставляет имитацию "сырого" ответа OpenSky API (список списков), включая одну заведомо некорректную запись (с пустым callsign и None в числовых полях) для проверки устойчивости парсера.

### test_aeroplane.py (Базовые тесты модели)

1. test_aeroplane_validation: Параметризованный тест. Проверяет, что при передаче некорректных данных (пустая строка, длина > 8, отрицательные значения, неверный тип) выбрасываются корректные исключения (ValueError или TypeError). 
2. test_aeroplane_to_dict: Проверяет, что метод to_dict возвращает словарь с правильными типами и округлёнными значениями. 
3. test_aeroplane_from_dict_success: Проверяет успешное создание объекта Aeroplane из полного валидного словаря. 
4. test_aeroplane_from_dict_missing_field: Проверяет, что при отсутствии обязательного поля (например, country) в словаре выбрасывается ValueError. 
5. test_aeroplane_compare_by_speed: Проверяет логику сравнения двух самолётов по скорости (возврат -1, 1, 0) и выброс TypeError при сравнении с объектом другого типа. 
6. test_cast_to_object_list_success: Проверяет, что метод корректно преобразует список словарей в объекты, игнорируя запись с пустым callsign.

### test_aeroplane_extended.py (Расширенные тесты модели)

1. test_aeroplane_validation_warnings: Проверяет, что при экстремальных, но допустимых значениях (скорость > 1000, высота > 15000) в лог записывается предупреждение (WARNING). 
2. test_aeroplane_compare_by_altitude: Проверяет логику сравнения двух самолётов по высоте и обработку TypeError. 
3. test_aeroplane_from_dict_with_optional_fields: Проверяет, что метод from_dict корректно обрабатывает и присваивает опциональные поля (on_ground, icao24, latitude, longitude). 
4. test_cast_to_object_list_handles_none: Проверяет, что при отсутствии значений (None) для скорости и высоты, они корректно заменяются на 0.0. 
5. test_cast_to_object_list_uses_geo_altitude: Проверяет, что если baro_altitude равен None, метод использует значение из geo_altitude. 
6. test_cast_to_object_list_skips_invalid_records: Проверяет, что метод пропускает записи с пустым callsign и записи с некорректными типами данных, не прерывая работу. 
7. test_aeroplane_str_representation: Проверяет, что метод __str__ возвращает строку, содержащую позывной, страну, скорость и высоту. 
8. test_aeroplane_repr_representation: Проверяет, что метод __repr__ возвращает строку технического представления объекта. 
9. test_aeroplane_to_dict_roundtrip: Проверяет цикл "сериализация -> десериализация": объект превращается в словарь и обратно, сохраняя все значения. 
10. test_aeroplane_callsign_validation: Параметризованный тест, проверяющий выброс исключений для некорректных позывных (пустая строка, длина > 8, не строка). 
11. test_aeroplane_coordinates_validation: Параметризованный тест, проверяющий выброс ValueError при выходе широты за пределы [-90, 90] или долготы за пределы [-180, 180].

### test_api_base.py (Тесты базового API)

1. test_make_request_success: Мокает requests.Session.request. Проверяет, что при успешном ответе сервера метод возвращает корректный распарсенный JSON и вызывает request с правильными аргументами. 
2. test_make_request_errors: Параметризованный тест. Проверяет, что при возникновении Timeout, HTTPError или RequestException метод возвращает None, а не роняет приложение. 
3. test_make_request_json_error: Проверяет, что при ошибке парсинга JSON (вызов ValueError внутри response.json()) метод возвращает None. 
4. test_api_repr: Проверяет корректность строкового представления (__repr__) тестового клиента.

### test_api_nominatim_extended.py (Расширенные тесты Nominatim)

### test_data_processing.py (Тесты утилит)

1. test_filter_aeroplanes: Параметризованный тест с 6 сценариями. Проверяет фильтрацию по стране, диапазонам высот, скоростей, статусу on_ground и отсутствие результатов. 
2. test_sort_aeroplanes: Параметризованный тест. Проверяет сортировку по разным полям (altitude, velocity, callsign) в прямом и обратном порядке. 
3. test_get_top_aeroplanes: Проверяет, что функция возвращает ровно N элементов, отсортированных по убыванию. 
4. test_get_aeroplanes_by_altitude_range: Проверяет работу обёртки над функцией фильтрации по высоте. 
5. test_print_aeroplanes_empty: Проверяет, что при пустом списке выводится сообщение "Нет данных", а заголовок приводится к верхнему регистру (проверка через .lower()). 
6. test_print_aeroplanes_with_data: Проверяет, что при наличии данных выводится таблица с заголовками и правильным количеством записей. 
7. test_show_status_bar: Проверяет, что функция выводит в консоль строку, содержащую переданные числа и слова "В памяти" / "В файле". 
8. test_filter_aeroplanes_no_criteria: Проверяет, что при отсутствии критериев функция возвращает копию исходного списка. 
9. test_filter_aeroplanes_multiple_criteria: Проверяет одновременную фильтрацию по нескольким условиям (страна + диапазон высот).

### test_storage_json_saver.py (Базовые тесты хранилища)

1. test_add_and_exists: Проверяет, что после добавления самолёта метод exists возвращает True, а count увеличивается.
