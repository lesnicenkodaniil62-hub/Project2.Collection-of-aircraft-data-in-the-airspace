# Проект: Сбор данных о воздушных судах в воздушном пространстве

## Описание

Сбор данных о воздушных судах в воздушном пространстве с использованием API OpenSky Network и Nominatim. Проект предоставляет инструменты для получения, фильтрации, анализа и сохранения телеметрии полётов с поддержкой системы снапшотов (версионирования данных), строгой типизацией и полным тестовым покрытием

## Использование:

Приложение управляется через интерактивное консольное меню. Следуйте этой подробной пошаговой инструкции:

### Шаг 1: Запуск приложения
Активируйте виртуальное окружение Poetry если оно не активно и запустите главный файл:
poetry run python main.py
После запуска вы увидите приветственный баннер и главное меню.

### Шаг 2: Получение данных
В главном меню выберите 
* Пункт 1. Получить самолёты по стране.
Введите название страны на английском языке (например, Spain, Germany или Russia).
Система автоматически найдёт координаты страны через Nominatim и запросит данные о самолётах через OpenSky API.

Важно: Если в оперативной памяти уже есть загруженные данные, система выведет диалог: "В памяти уже есть X самолётов. Что сделать? (1) Заменить текущие данные (2) Добавить к текущим". Введите 1 или 2 в зависимости от вашей цели.

### Шаг 3: Анализ и фильтрация данных
Используйте следующие пункты меню для работы с загруженным набором данных:
* Пункт 2 (Топ N по высоте): Введите целое положительное число N (от 1 до общего количества самолётов). Система отсортирует данные по убыванию высоты и покажет таблицу с топ-N самолётами.
* Пункт 3 (Фильтр по стране): Введите названия стран через запятую (например: Spain, Germany). Система отфильтрует список, оставив только самолёты, зарегистрированные в этих странах (регистр не важен).
* Пункт 4 (Фильтр по диапазону высот): Введите диапазон в строгом формате min-max (например: 5000-10000). Система покажет только те самолёты, чья высота попадает в этот интервал.
* Пункт 5 (Показать все): Выведет полную отформатированную таблицу со всеми самолётами, находящимися сейчас в оперативной памяти.

### Шаг 4: Сохранение и загрузка (Система снапшотов)
* Пункт 6 (Сохранить в снапшот): Создаёт новый файл в папке data/ с уникальным именем, содержащим дату и время (например, aeroplanes_20231025_143000.json). Это позволяет хранить историю изменений без перезаписи старых файлов.
* Пункт 7 (Загрузить из снапшота): Показывает нумерованный список всех доступных снапшотов. Введите номер нужного файла. Внимание: если в памяти есть несохранённые изменения, система выдаст предупреждение и попросит подтвердить перезапись данных.

### Шаг 5: Завершение работы
* Выберите пункт 0. Выход. Если в оперативной памяти есть данные, которые не были сохранены в снапшот, программа вежливо предупредит вас: "В памяти есть несохранённые данные. Выйти без сохранения? (y/n)". Введите y для выхода или n для возврата в меню.
